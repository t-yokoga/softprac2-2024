    課題 R1 「TOKYO PARKING」

    課題の概要

    この課題では，TOKYO PARKING（駐車場パズル）を C プログラム tokyoparking.c として実装することを目的とする．

    TOKYO PARKING

    TOKYO PARKING のルールは以下の通りである．

    • 開始時には，ボード上に決められた配置で車（のコマ）が置かれている
    • 出口の場所は決められている
    • 定められた車を出口まで動かすとクリアとなる
    • 車は縦または横向きに配置されていて，前後にしか動けない
    • 車の向きを変えることはできない
    • 既に他の車がいる場所には動けない

    この課題では，標準入出力を用いて，車と移動方向の指定とボードの表示を行う． 「移動させる車の指定」→「移動方向の指定」という手順で入力を行い，更新されたボードの情報を出力する． したがって，このプログラムを実行したときの標準入出力上の表示は以下のようになる．

    処理の流れ

    TOKYO PARKING を実現するプログラムの処理の流れは以下の通りとなる．

    1. 初期配置を定義する
    2. 現在のボードの状態を表示する
    3. 操作として「動かす車と向き」を標準入出力から読み取る
    4. 可能な動作であるか判定する
      1. 可能な動作であるならば，5へ
      2. 可能な動作でないならば，1へ
    5. 操作に従ってボードを更新する
    6. 目的の車が出口に到達しているか判定する
      1. 到達していれば，ゲームクリアとしてプログラムを終了する
      2. 到達していなければ，2へ

    これを踏まえて，TOKYO PARKING を C プログラムとして実装する．

    データ構造

    まずはデータ構造を決める必要がある． ボードは，整数型の二次元配列board[HEIGHT+2][WIDTH+2]として定義する． パズルの縦横のマス目の数HEIGHTおよびWIDTHは，#defineを用いてマクロ定義する． （今回はボードのサイズは６×６としておく．）

    board[i][j]の値は，ボードの左から i 番目，上から j 番目のマスの状態を表しており，それぞれ以下のように対応づけることにする．

    • 0：車が配置されておらず，移動可能なマス
    • 1, 2, ...（車に対応した番号）：既に車が配置されており，移動不可能なマス
    • -1：ボード外周の移動不可能なマス

    ここで車に対応づける番号は，出口へと移動させるべき車を1とし，それ以外の車を2, 3, ... と割り当てることにする． 例として，図1のボードに対応する配列boardの要素の値は図2のようになる．


    図2：初期配置に対応した配列 board の要素の値

    また，車が縦向きか横向きかのいずれかによって，移動可能な方向は異なる． ゲームのプレイ中に車の向きが変わることはないため，車の向きの情報はグローバルに定義された配列is_vertical[MAX+1]へとあらかじめ格納しておくことにしよう． ここでMAXは車の台数とし，#defineを用いてマクロ定義しておく． is_vertical[i]の値は車iが縦向きなら1，横向きなら0とする． さらに初期配置についても，二次元配列 init[HEIGHT+2][WIDTH+2] としてグローバルに定義しておく．

    無効な操作の判定

    現在のボードの配置によっては無効となる操作が存在する． 具体的には，車の向き（縦・横）と方向が一致しない場合や，外周や他の車とぶつかってしまう場合は操作が無効となる． 無効な操作に基づいてボードを更新することはできないため，更新前に操作が有効であるか無効であるかを判定しないといけない．

    無効となる入力の検出

    • 車の向きと方向が一致しない場合の検出
      • 車の向きの情報が格納された配列 is_vertical を参照すれば検出できる
    • 外周や他の車とぶつかってしまう場合の検出
      • 以下の手順で検出できる
        1. board を探索して車の両端の場所を探す
          • 縦なら (i, j) と (k, j)
          • 横なら (i, j) と (i, k)
        2. board の移動先の場所の要素が 0 かどうかを調べる
          • 移動方向が上や左なら board[i-1][j] やboard[i][j-1] を見る
          • 移動方向が下や右なら board[k+1][j] やboard[i][k+1] を見る

    ボードの更新

    有効な操作が入力された場合は，操作に従ってボードを更新することになる． このとき，移動先となるマスと，車の最後尾のマスの二カ所を変更しないといけない． そのため，まずボードを探索して移動させる車の両端の位置を求めた上で，移動方向をもとに変更するマスの位置を計算し，更新を行う． 具体的には，以下のような手順となる．

    1. board を探索して車の両端の場所を探す
      • 縦向きの場合，それぞれ (i, j) と (k, j) とする
      • 横向きの場合，それぞれ (i, j) と (i, k) とする
    2. 移動方向が
      • 上なら，board[i-1][j] と board[k][j] を入れ替える
      • 左なら，board[i][j-1] と board[i][k] を入れ替える
      • 下なら，board[i][j] と board[k+1][j] を入れ替える
      • 右なら，board[i][j] と board[i][k+1] を入れ替える
