    課題 R-3 「辞書プログラムの作成」

    課題の概要

    単語の名前を入力すると意味を出力する辞書の機能をもつプログラムdic.cを作成せよ． 辞書データは名前と意味の2つをもち，データの追加・削除が可能である． 辞書機能を実現するためにはデータの追加・削除・検索が可能なデータ構造を実装する必要があるが，本課題では，線形リストと二分探索木という2通りのデータ構造を用いる．

    インターフェース

    このプログラムは実行されると入力待ち状態となり，単語が入力されたらそれに対応する内容を出力するものとする． ただし，以下の文字列が入力された場合は，それぞれに応じた辞書の操作を行うものとする（始めに-をつけることで，単語と区別する）．

    1. -add ：単語の追加
    2. -delete ：単語の削除
    3. –quit ：プログラムの終了

    実行例を以下に示す．

    アルゴリズム

    このプログラムの実行の流れは以下のようになる．

    1. 標準入出力からの入力を待つ
    2. 入力に応じて以下の処理を繰り返す
      1. -addなら，単語の追加処理を行う
      2. -deleteなら，単語の削除処理を行う
      3. -quit なら，プログラムを終了する
      4. それ以外なら，その単語で辞書を検索する
        1. 単語が見つかれば，その単語の意味を出力する
        2. 単語が見つからなければ，見つからなかったことを知らせる
      5. 2へ戻る

    課題1 線形リストを用いた辞書機能の実現

    この課題では，随時データの追加・削除を行うため，データ構造として構造体とポインタによる線形リストを用いる．

    線形リスト

    線形リストは，図1のように構造体をポインタで直線的に接続した構造をもつ． リストの各要素は，key と next という（少なくとも）2つのメンバをもつ構造体 Node として定義する．


    図1：線形リスト

    keyはリストの各要素がもつデータを表す．nextはリストの要素を指すポインタであり，リストを連結するために用いる． リストの先頭は Node * 型のポインタ head で表しており，リストの最後の要素がもつポインタは NULL を指している．

    int型のデータをもつリストにおけるNodeの定義例を以下に示す．

    1
2
3
4
    typedef struct node{
    int key;
    struct node *next;
} Node;

    線形リストへのデータの追加，探索，削除はそれぞれ以下に示す手続きによって実現することができる．

    データの追加

    線形リストへデータ val を追加する場合，図2に示すように新たな要素となる Node 型の構造体を作成してその key を val とし，リストの先頭に付け加える．


    図2：線形リストへのデータの追加

    動的なデータの作成

    • データの作成には malloc 関数を用いる．この際に #include <stdlib.h> する必要がある． 
      1
2
      Node *new;
new = (Node *)malloc(sizeof(Node));
      malloc 関数は，戻り値として新たに作成した構造体へのポインタを返す． 上記の例では，ポインタを変数 new へ格納している．

    これを実現するための手順は，以下の通りとなる．

    1. malloc 関数で新たな構造体を作成し，その位置を指すポインタを new とする．

    2. new->key に値 val をコピーする．

    3. new->next に，ポインタ head をコピーする．

    4. head に newをコピーする．

    データの探索

    線形リストの探索では，図3に示すようにリストから値 val をもつ構造体を探索し，その位置をポインタとして返す．


    図3：線形リストのデータの探索

    求めるポインタを p とすると，この処理は以下の手順で実現できる．

    1. p を head で初期化する．

    2. p != NULL である間，以下の処理を繰り返す：

      1. p が指す要素の値（p->key）と val を比較する．
        • 等しくなければ，次の要素を調べるため p を p->next へと更新する．
        • 等しければ，p を求めるポインタとして処理を終了する．

    3. p == NULL となった場合は，リストを最後まで探索しても求める要素が存在しないため，手続きを終了する．

    データの削除

    線形リストからデータ val を削除する場合，図4に示すようにリストから値 val をもつ要素 target を探索し，要素 target の直前の要素と直後の要素をポインタ next で接続した上で要素 target を削除する．


    図4：線形リストのデータの削除

    動的なデータの削除

    • データの削除には free 関数を用いる．malloc と同様に #include <stdlib.h> する必要がある． 
      1
2
      Node *new;
free(new);
      free 関数は引数として，削除対象となるデータへのポインタをとる． 上記の例では，ポインタ new が指す構造体のデータを削除している．

    ここで，target を削除するためには target を指すポインタを参照する必要があり，その直前の要素と直後の要素を接続するにはtargetの直前の要素の next を更新する必要がある． そのため，以下の 2 つのポインタ Node **p および Node *q を用いる．

    • p は探索中の要素を指すポインタへの二重ポインタであり，targetの直前の要素のnextを更新するために用いる．
    • q は要素targetを指すためのポインタであり，targetを削除するために用いる．

    そして，以下の手順で val をもつ要素の削除が実現できる．

    1. p を head へのポインタで初期化する（p=&head）．

    2. *p != NULL である間，以下の処理を繰り返す：

      1. *p の指す要素の値 (*p)->key が val と等しいか調べる．

        • 等しければ：

          1. *p は target を指しているため，q に *p をコピーする．

          1. *p を target の next（つまり (*p)->next）で更新する．

          1. q が指す要素を削除する．

        • 等しくなければ，次の要素を調べるためpを「*pが指す要素のnextへのポインタ（つまり&((*p)->next)）」へ更新する．

    3. *p = NULL となった場合は，リストを最後まで探索しても求める要素 target がリスト上に存在しないため，手続きを終了する．

    ポインタ関連の演算子の優先順位

    • 「ポインタが指す～」を表す演算子 * や「〜のアドレス」を表す演算子 & と，「ポインタが指す構造体のメンバ～」を表す演算子 -> では，-> の優先順位が高い．
    • なので，「二重ポインタ p が指しているポインタが指す構造体の要素 key 」を表現するには，括弧を付けて (*p)->key とする必要がある．

    データ構造

    このプログラムでは，各単語のデータを以下のメンバをもつ構造体で表し，その上で線形リストを構成する．

    • id ：名前．他とは重複しない． 文字数は半角16文字以下とする．
    • def ：意味．他と重複してもよい．文字数は半角256文字以下とする．
    • next ：リストの次要素へのポインタ．

    辞書の単語を表し，線形リストを構成する要素となる構造体 Word を以下のように定義する．

    1
2
3
4
5
    typedef struct word{
    char id[16];
    char def[256];
    struct word *next;
} Word;

    単語の追加

    次に，リストへ単語を追加するための機能を関数 add_word() として実装する． データの追加 で述べた通り，新たなデータはリストの先頭に追加する． ここでは，名前の重複の可能性については一旦考えないこととして，与えられたデータをリストの先頭へ機械的に追加することにしよう．

    なお，リストの先頭にデータを追加するため，リストの先頭へのポインタ head の値を変更する必要がある． 関数内で head を変更する方法としては，以下の 2 つがある．

    • 関数の戻り値として新たな先頭へのポインタを返して，変数 head に代入する．
      • head = add_word(...);
    • 関数の引数として head のアドレスを渡して，関数内で値を更新する．
      • add_word(&head, ...);

    ここでは，前者の方法を採ることにする．

    演習課題2

    名前を表す文字列 id，意味を表す文字列 def，単語が格納されている線形リストの先頭へのポインタ head を引数とし，単語を新たにリストの先頭へと追加した上でリストの先頭の要素へのポインタを戻り値として返す関数 add_word() を作成せよ．

    プロトタイプ宣言

    head = add_word(new_id[1], new_def[1], head);
    head = delete_word("orange", head);

    temp = search_word("apple", head);
    if (temp != NULL){
        print_word(temp);
    } else {
        printf("「apple」は登録されていません．");
    }
    temp = search_word("orange", head);
    if (temp != NULL){
        print_word(temp);
    } else {
        printf("「orange」は登録されていません．");
    }
}

    このサンプルコードでは，単語 apple と orange を add_word() 関数によってリストに登録した後で，orange を削除している． その後，search_word() 関数で単語 apple と orange を検索し，リストに存在するならその結果を，print_word() 関数によって表示し，存在しないならその旨を表示している．

    課題2 二分探索木を用いた辞書機能の実現

    この課題では，随時データの追加・削除を行うため，データ構造として二分探索木を用いる．

    二分探索木

    二分探索木は図5に示すように，それぞれの頂点が左右2つの子をもつ構造となる．


    図5：二分探索木

    二分探索木をCで実装する場合，それぞれの要素 Node をデータ key と ポインタ left，right という3つのメンバをもつ構造体として定義する． left および right は Node を指すポインタであり，二分探索木の要素を連結するために用いる．

    int 型のデータをもつ二分探索木であれば，Node は以下のように定義される．

    二分探索木の要素を表す構造体 Node の定義
    1
2
3
4
    typedef struct node{
    int key;
    struct node *left, *right;
} Node;

    二分探索木のすべての頂点 は，以下の関係を満たす必要がある．

    • 左の部分木には， より小さい要素のみが含まれる．
    • 右の部分木には， より大きい要素のみが含まれる．

    図5の二分探索木は 1, 3, 4, 8 の 4 つの値をもつが，図6(a)に示すように，値 3 をもつ頂点 の左部分木には より小さい値 1 が，右部分木には より大きい値 4 と 8 が含まれている． この関係は部分木内の頂点間でも同様に成り立っており，図6(b)に示すように，値 8 をもつ頂点の左部分木に，より小さい値 4 が含まれている．



    (a)


    (b)
    図6：二分探索木の頂点間の関係

    二分探索木に対してデータの追加および削除を行う際には，上記の関係が保存されていなければならない．

    二分探索木のデータの探索

    上述のとおり，この関係は部分木内の頂点でも同様に成り立つため，二分探索木に対して要素の探索を行う際には，現在参照している頂点の値 key が求める要素 val でなかった場合，val > key ならば左の部分木，val < key ならば右の部分木に対してのみ探索を行えばよい． このように，探索の際に全ての要素をチェックする必要がないため，計算量を少なくできるのが二分探索木の利点である．

    二分探索木へのデータの追加

    二分探索木に新たな要素 val を追加する際には，探索と同様に val と現在の頂点の key を比較し，val > key ならば左の部分木，val < key ならば右の部分木に対して再度探索を繰り返す． そして，最終的に探索先が NULL となったとき，その場所が要素 val を追加すべき場所となる．

    例として，図5の二分探索木に新しく要素 6 を追加する場合を考える．

    (b)
    図7：二分探索木へのデータの追加

    図7(a)に示すように，各頂点との比較を繰り返した結果，追加すべき場所は要素 4 の右の子要素となる． 要素 6 を追加した後の二分探索木の状態は図7(b)のようになる．

    二分探索木のデータの削除

    二分探索木からデータを削除するための処理は，線形リストに比べて少し複雑である． 削除の際の処理は，削除対象の要素 val をもつ頂点（以下，簡単に頂点 という）の子の個数によって異なる．

    • 頂点 が子をもたない場合

      • 頂点 を削除するだけでよい．

    • 頂点 が子を一つだけもつ場合

      • 頂点 を削除した上で，頂点 へのポインタを頂点 の子へ繋ぐ．

    • 頂点 が子を二つもつ場合

      • 頂点 の右の部分木で最も小さい値をもつ頂点 を頂点 の代わりに配置する．

        このような頂点 は，頂点 の右の部分木を左へ左へと探索することで見つけることができる． 頂点 は頂点 の左の部分木のどの要素よりも大きく，右の部分木のどの要素よりも小さいため， を の代わりに配置することで二分探索木の条件を保つことができる．
